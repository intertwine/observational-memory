# v0.8.0 Plan — Memory Durability, Provenance & Recall Quality

Status: **ACTIVE** (scope locked 2026-05-31). Gate-by-gate, milestone discipline.
Design rationale: [`plans/v0.8.0-design-decision.md`](v0.8.0-design-decision.md).
Issue: [#71](https://github.com/intertwine/observational-memory/issues/71) (umbrella).
v1.0 trajectory: [`plans/team-memory-direction.md`](team-memory-direction.md).

## What v0.8.0 is — and is not

v0.7.0 (M2+M3, section-targeted reflection) solved the acute scaling problem #71
was filed for: the O(chunks × document size) whole-document resend is gone.

After a design-vetting pass and a grill-me review, **the full addressable
memory-unit store (the old plan's Milestones 4–8) is deferred indefinitely and
data-gated.** Post-M3 it is internally contradictory — an authoritative unit
store with supersession edges cannot coexist with "Markdown is the source of
truth," which is the product's core advantage — and it targets residuals (fat
sections, sub-fact addressability) that are not real at personal scale (the real
corpus is ~37 KB). The five M4-class open questions (SQLite vs JSONL, opt-in
cadence, cluster unit-sharing, metadata visibility, `om memory` CLI) are
therefore **moot — no store is being built.**

Instead, v0.8.0 spends its slot on three things at once: **durability** (a memory
backup path), **honesty/provenance** (where memory came from, plus the v1.0
governance seed), and **recall quality** (where users actually touch the
product). Provenance is the single primitive that serves both #71's "traceable to
source" criterion and the v1.0 §6.3 door-opener — one investment, two roadmaps.

## Principles (unchanged)

- **Markdown stays authoritative.** Inline metadata is the source of truth; typed
  copies may ride on retrieval objects. No second authoritative datastore.
- **Fail closed.** Invalid input leaves memory unchanged; never write a partial or
  corrupt document.
- **Smallest reversible step.** Every gate is additive, default-preserving, and
  independently revertible.
- **Gate discipline.** Implement the next gate only; adversarial review (≥5
  failure modes); then STOP for Bryan's end-to-end walkthrough and explicit
  acceptance before the next gate. Fan out agents *within* a gate, never across.
- **Honor the cluster rules.** Never sync the live data dir; off-host is a future
  goal reached through explicit, portable bundles, not live-dir replication.

## Gates (sequenced)

Backup goes first deliberately: once memory can always be rolled back, every later
gate operates on a recoverable asset.

### Gate 1 — Host-local memory backup *(first; in progress)*

Defend against **`om` self-corruption and accidental deletion**. (Machine-loss /
off-host is an explicit **future** goal, not built now — but the bundle format
must not foreclose it.)

- **Local versioned snapshots** of the memory files (reuse `_backup_existing_memory`).
- **Automatic pre-reflect snapshot** of the last-good state before each reflect
  write — this is the load-bearing piece; it lets a bad reflect be rolled back and
  is what makes deferring compaction safe.
- **Explicit `om backup`** (on demand) and **`om restore`** (from a chosen
  snapshot), plus rotating retention.
- **Portable, self-contained bundle format** a future off-host transport can ship
  as-is. No off-host automation now.
- Acceptance: a corrupted/deleted `reflections.md` is fully recoverable from a
  snapshot; pre-reflect snapshot is taken on every reflect; retention bounds disk;
  restore is byte-faithful; nothing writes into or syncs the live data dir
  improperly; fail-closed on a bad/missing snapshot.
- Adversarial focus: data-loss/corruption safety, byte-faithful restore, disk
  growth/retention, cluster-rule compliance (no live-dir sync), fail-closed
  behavior, CLI/UX clarity, test adequacy.

### Gate 2 — Talk recall robustness *(this is the v0.8.0 gate; PR #84)*

The talk-robustness work is the v0.8.0 Gate 2. The recall-quality **benchmark**
was split out as a separate, independently-tracked unit (Gate 2b, below) — it
lives in another repo and does not gate the v0.8.0 sequence.

- Distinguish "recall timed out / backend unavailable" from "no relevant memory
  found" in both the spoken turn and `--json` (propagate through **every** recall
  consumer, not just talk).
- `OM_TALK_RECALL_TIMEOUT` env knob (replace the hardcoded 8s in
  `Conversation._recall_for`); fail closed on unset/garbage.

### Gate 2b — Recall-quality benchmark *(separate unit, NOT a v0.8.0 gate)* (#81)

Tracked independently in `observational-memory-bench`; decoupled from the v0.8.0
gate sequence and acceptance. Build a labeled query→expected-doc recall
**benchmark** (n≥5 seeds, bootstrap CIs, Holm-corrected permutation tests per
repo rules) comparing `bm25`/`qmd`/`qmd-hybrid`/`moss` on om's own corpus. Its
results inform a future (post-v0.8.0) decision on backend defaults / fusion.

### Gate 3 — Section-level rot-proof provenance

- Stamp `last_reflected` + `derived_from_obs_window=<date-range>` per section via
  the existing post-hoc `ensure_reflection_metadata` pass (NOT per-fact
  `source_obs` pointers — those dangle when observations age out).
- Carry typed `owner`/`scope`/`source` on `Document`/`SearchResult` via the
  `moss.py` inline↔typed round-trip.
- Meets #71's "traceable to source" honestly; seeds v1.0 §6.3.

### Gate 4 — Generalize the scope-filter mechanism

- Evolve `filter_reflection_entries_for_cluster` into a pluggable scope-aware
  pipeline (the v1.0 §6.2 governance seed) — **mechanism only; add no inert
  `team`/`org` values** to the leak-critical filter until a tier consumes them.
- DONE: shipped as `SHAREABLE_SCOPES = frozenset({"cluster"})` + the pure
  `_scope_is_shareable(scope)` resolver (the §6.2 governance-seed socket).
  Default-deny on explicit-unknown scopes (typo/hallucinated/future/hand-typed
  `team`/`org` all fail closed); absent scope rides along (structure, prose,
  stamps, unstamped bullets). Both share-out paths route through the one
  resolver: `filter_reflection_entries_for_cluster` (cluster snapshots) and
  `moss._strip_local_lines` (cloud upload). No inert `team`/`org` values added;
  a future tier widens sharing by adding a member to `SHAREABLE_SCOPES`.

### Gate 5 — Conflict surfacing (read-only) — ACCEPTED (Bryan, 2026-06-10)

- `om reflect --check-conflicts` diffs the prior `reflections.md` against the
  **final reassembled** new document (never per-fold — per-fold reintroduces the
  O(folds × doc) shape M3 killed) and surfaces high-stakes facts a single reflect
  cycle silently changed. Emits a throwaway review artifact
  (`om-conflicts-latest.md` in the temp dir, overwritten, never synced); never
  mutates durable Markdown; always exits 0. `--json` for machines.
- Design note: `find_reflection_conflicts` is cross-host (needs ≥2 records) and
  returns nothing over one host's single document, so Gate 5 adds the single-host
  complement `diff_reflection_conflicts(prior, new)`. It shares the reviewability
  gate (`_is_reviewable_entry`) with the cluster function — single source of
  truth — and fires three precision-first signals: explicit-id divergence,
  one-to-one (section, kind) slot divergence, and a **prior-anchored** solo-section
  downgrade catch (so a new-side reclassification can't hide a reworded
  guardrail). Comparison is normalization-tolerant (bold/quotes/whitespace/
  trailing punctuation ignored). Known residual (deferred, precision over recall):
  a high-stakes fact reworded inside a multi-bullet section while its kind also
  changes, and silent drops, are not flagged.

### Gate 6 — B0 growth measurement (read-only) — ACCEPTED (Bryan, 2026-06-10)

- Section/H3 size + coldness measurement in `om doctor` / `--quality-report`.
  Pure, read-only. Run it for a release so any future compaction decision is
  data-grounded, not speculative.
- Shipped as `growth.measure_memory_growth(config)`: per-document
  bytes/lines/bullets for `reflections.md`/`profile.md`/`active.md`/
  `observations.md`, per-H2/H3 bytes/lines/bullets/share of `reflections.md`,
  plus top-N fattest and coldest sections and simple totals. Coldness is the
  age of the newest timestamp recoverable from each section's own content —
  Gate 3 `<!--om-section: last_reflected=… derived_from_obs_window=…-->`
  stamps, per-bullet `last_seen`/`last_verified`, and dated headings; a section
  with no recoverable timestamp reports `age_days: null`, never a guess.
- Surfaced in both plan-named places: `om doctor` gets a compact
  "Memory growth (B0)" block (total size, largest section, coldest section)
  that degrades to a single WARN and can never fail doctor, and
  `om context --quality-report` carries a `growth` key in JSON plus a readable
  text section. Strictly read-only and fail-soft: missing files yield an
  empty-but-valid report, non-UTF-8/malformed content is measured best-effort,
  and the entry point never raises.
- Walkthrough evidence (Bryan's live corpus, 2026-06-10): `om doctor` reported
  reflections.md 45.3 KB across 9 sections / 9 subsections / 93 bullets,
  largest section "Active Projects" at 15.5 KB (34%), coldest section 0d old,
  total memory 428.3 KB — growth is real but no compaction threshold is near,
  which is exactly the instrument-don't-act posture B0 was built for.

## Explicitly OUT of v0.8.0

- Full memory-unit store (deferred indefinitely; revisit only on Gate-6 data).
- Destructive hierarchical compaction (deferred behind Gate-6 data).
- Multi-backend recall fusion seam (the #81 benchmark decides next release whether
  merging backends is worth it — no speculative plumbing).
- Off-host / automated backup replication (future goal; the bundle format keeps
  the door open).
- Per-fact provenance pointers (rot by design).
- [#54](https://github.com/intertwine/observational-memory/issues/54) Obsidian
  backing store (backlog).
- [#77](https://github.com/intertwine/observational-memory/issues/77) async-Batch
  provider/model validation (independent P3; can ride any release).
